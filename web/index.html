The three questions

Why · What if · Should we

Most map tools stop at "what's here." SIMyCity is built around the questions a resident, a developer, or a city actually asks.

01 — WHY

Why isn't there a Costco near me?

Not income — it's drive-time rooftops, road networks, and who already serves the trade area. And why aren't developers building the homes that would change that? simy standoffs

02 — WHAT IF

Would a data center succeed on this plot?

It needs power, water, and fiber nearby — and induces schools, fire, and roads at scale. SIMyCity walks the dependency graph. simy use data_center

03 — SHOULD WE

Should it be built at all?

The same parcel looks different to a developer, a resident, an environmentalist, and the city. We hold those in tension instead of one score. simy perspectives

How it works

Dependencies & standoffs, modeled like SimCity

Every land use draws on service layers and induces new demand. And uses gate each other — creating chicken-and-egg standoffs that are often the real reason "nothing is here."

Semi-automated

Contribute while you sleep 🌙

Got Claude Code capacity sitting idle overnight? Point it at a pre-scoped task and wake up to a pull request. Humans decide what and whether; agents do a lot of the how — and every change still lands through review.

Guardrails: agents propose, humans merge · no secrets in the repo · scope fenced to the task's files · schema tests are the automatic backstop.

  1. Pick a plotan agent-ready issue with files, interface, and acceptance test spelled out.
  2. Point your spare tokens at itrun an agent in the repo while you're away.
  3. It opens a PRtests + registry validation run automatically in CI.
  4. You review & mergethe garden grows a patch — credited to you.
Why we're doing this

An honest tool, not a hype machine

Closed, well-funded versions of "model the built world" exist. The public-good version — open, free-data-first, owned by the people whose communities it describes — doesn't yet. That's the gap.

Understand the place around you. Then help make it better — digitally, and in the real world.
